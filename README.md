# Biblioteka

`Biblioteka` - настольное WPF-приложение для учета небольшой библиотеки. Программа хранит читателей, авторов, издания, физические экземпляры книг, выдачи, возвраты, список должников, отчеты и каталожную карточку выбранной книги.

## Стек

- `.NET 8`
- `WPF`
- `Microsoft.Data.SqlClient`
- `SQL Server / SQL Server Express`

## Возможности

- учет читателей: поиск, добавление, редактирование и удаление
- учет авторов с быстрым добавлением из формы книги
- каталог изданий с фильтрами по названию, автору, жанру и году
- учет экземпляров с инвентарным номером, местонахождением и статусом
- оформление выдачи с датой выдачи и сроком возврата
- оформление возврата с автоматическим изменением статуса экземпляра
- список должников по просроченным активным выдачам
- сводная панель с ключевыми показателями
- отчеты по книгам, доступным экземплярам, выданным книгам и должникам
- каталожная карточка выбранного издания

## Структура проекта

- `Biblioteka/` - WPF-приложение
- `Biblioteka/Data/LibraryRepository.cs` - доступ к SQL Server и бизнес-правила
- `Biblioteka/MainWindow.xaml` - главное окно и интерфейс разделов
- `Biblioteka/App.config` - строка подключения к базе данных
- `Database/LibraryDB.sql` - создание структуры базы
- `Database/LibraryDB.Seed.sql` - заполнение демонстрационными данными

## Требования

- Windows
- `.NET SDK 8.0` или новее
- `SQL Server` или `SQL Server Express`
- права на создание и использование базы `LibraryDB`

## Подготовка базы данных

1. Выполните `Database/LibraryDB.sql` для создания структуры БД.
2. Выполните `Database/LibraryDB.Seed.sql` для заполнения демонстрационными данными.

Пример через `sqlcmd`:

```powershell
sqlcmd -S .\SQLEXPRESS -E -i Database\LibraryDB.sql
sqlcmd -S .\SQLEXPRESS -E -i Database\LibraryDB.Seed.sql
```

Важно:

- `LibraryDB.sql` пересоздает схему и удаляет существующие таблицы `Loans`, `BookCopies`, `Books`, `Readers` и `Authors`.
- `LibraryDB.Seed.sql` добавляет демонстрационных авторов, читателей, книги, экземпляры и историю выдач.

## Настройка подключения

Приложение читает строку подключения из `Biblioteka/App.config`.

Пример строки подключения из проекта:

```xml
<add name="LibraryDB"
     connectionString="Data Source=DESKTOP-ROO0H74\SQLEXPRESS; Database=LibraryDB; Integrated Security=True; Persist Security Info=False; Pooling=False; MultipleActiveResultSets=False; Encrypt=True; TrustServerCertificate=True; Command Timeout=0"
     providerName="Microsoft.Data.SqlClient" />
```

Перед запуском замените `Data Source` на имя своего экземпляра SQL Server, если оно отличается.

## Запуск

Через IDE:

- откройте `Biblioteka.sln`
- назначьте проект `Biblioteka` стартовым
- запустите приложение

Через CLI:

```powershell
dotnet restore Biblioteka.sln
dotnet build Biblioteka.sln
dotnet run --project Biblioteka\Biblioteka.csproj
```

## Основные разделы интерфейса

- `Главное` - сводка и быстрые действия
- `Читатели` - работа с карточками читателей
- `Издания` - каталог книг и связь с авторами
- `Экземпляры` - учет фонда и статусов
- `Выдача` - регистрация новых выдач
- `Возврат` - оформление возврата книги
- `Должники` - просроченные активные выдачи
- `Отчеты` - набор табличных отчетов
- `Каталожная карточка` - библиографическая карточка выбранной книги

## Бизнес-правила

- ФИО читателя и номер читательского билета обязательны
- название книги и автор обязательны
- инвентарный номер и местонахождение экземпляра обязательны
- статус экземпляра может быть только `доступна`, `выдана` или `потеряна`
- выдать можно только экземпляр со статусом `доступна`
- срок возврата не может быть раньше даты выдачи
- дата возврата не может быть раньше даты выдачи
- один экземпляр не может иметь больше одной активной выдачи
- номер билета и инвентарный номер должны быть уникальными

## Демоданные

После выполнения `LibraryDB.Seed.sql` база содержит стартовый набор данных для быстрой проверки:

- 12 читателей
- 10 авторов
- 12 изданий
- 24 экземпляра
- историю выдач с возвращенными и активными книгами

## Примечания

- Целевая платформа проекта - `net8.0-windows`, поэтому приложение предназначено только для Windows.
- Ошибки SQL Server и валидации показываются пользователю в диалоговых окнах.
